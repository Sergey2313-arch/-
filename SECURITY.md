# Security Policy — ReviMarket

Этот документ описывает базовые правила безопасности для проекта **ReviMarket**.

## Текущий статус

Сейчас проект является **frontend beta**:

- нет настоящей регистрации;
- нет настоящих паролей;
- нет настоящих платежей;
- демо-данные могут храниться в `localStorage` только после согласия пользователя;
- банковские карты, CVC, паспортные данные и реальные платёжные реквизиты не хранятся.

## Что нельзя хранить в репозитории

Никогда не добавлять в GitHub:

- пароли от базы данных;
- строки подключения с настоящими логинами;
- секретные ключи ЮKassa / CloudPayments / Robokassa;
- SMTP-пароли;
- JWT-secret;
- приватные ключи;
- персональные документы пользователей;
- выгрузки реальной базы данных.

Для секретов использовать:

- `appsettings.Development.json` локально;
- User Secrets в ASP.NET Core;
- Environment Variables на хостинге;
- GitHub Secrets для GitHub Actions.

## Минимальная защита для backend-версии

Когда появится ASP.NET backend, обязательно добавить:

- ASP.NET Core Identity;
- хеширование паролей;
- роли `Customer`, `Designer`, `Admin`;
- проверку владельца заказа;
- защиту форм через AntiForgery Token;
- HTTPS;
- ограничение загрузки файлов;
- логирование действий без лишних персональных данных;
- хранение платежей только через `ProviderPaymentId`, без данных карты;
- страницу политики персональных данных;
- согласие на обработку персональных данных.

## Проверка доступа к заказам

Пользователь не должен видеть чужие заказы даже по прямой ссылке.

Пример логики для ASP.NET:

```csharp
if (order.CustomerId != userId && order.DesignerId != userId && !User.IsInRole("Admin"))
{
    return Forbid();
}
```

## Загрузка файлов

Разрешённые типы для beta/MVP:

- `.jpg`
- `.jpeg`
- `.png`
- `.webp`
- `.pdf`

Правила:

- максимальный размер файла: 10–25 МБ;
- не использовать оригинальное имя файла на сервере;
- сохранять файл через случайный GUID;
- не давать пользователю загружать `.exe`, `.js`, `.bat`, `.cmd`, `.php`, `.html`.

Пример безопасного имени:

```text
/uploads/orders/8f3a9c21-92aa-4b3d-9e90-a13c2f2d91c0.png
```

## Платежи

В проекте нельзя хранить:

- номер карты;
- CVC;
- срок действия карты;
- паспорт;
- банковские реквизиты без необходимости.

В базе хранить только техническую информацию:

```text
PaymentId
OrderId
UserId
Amount
Commission
Status
ProviderPaymentId
CreatedAt
```

## Инциденты безопасности

Если найден баг безопасности:

1. Не публиковать реальные данные пользователей.
2. Создать приватную заметку/issue без секретов.
3. Исправить проблему в отдельной ветке.
4. Проверить, что секреты не попали в историю Git.
5. Если секрет попал в Git — заменить его, а не просто удалить файл.

## Чеклист перед реальным запуском

- [ ] Нет секретов в GitHub.
- [ ] Включён HTTPS.
- [ ] Пароли хешируются через ASP.NET Identity.
- [ ] Есть роли пользователей.
- [ ] Заказы защищены проверкой владельца.
- [ ] Админка доступна только роли `Admin`.
- [ ] Файлы проходят проверку размера и расширения.
- [ ] Платежи идут через провайдера.
- [ ] Реальные данные карты не хранятся.
- [ ] Есть политика персональных данных.
- [ ] Есть пользовательское соглашение.
- [ ] Есть логирование действий.
- [ ] Логи не содержат пароли, токены и приватные сообщения целиком.
